# Repository Context Workflow

- Start each task by reading `SYSTEM_CONTEXT.md`. It is the one-stop architecture and workflow snapshot for this repo.
- `SYSTEM_CONTEXT.md` is generated by `pnpm context:update`. Do not edit it manually.
- If you change routes, shared schemas, store shape, API handlers, build scripts, dependencies, or user-facing flows, update `docs/context-notes.md` if needed and run `pnpm context:update` before finishing.
- Keep `docs/context-notes.md` focused on non-obvious product rules, architectural decisions, and known gaps that cannot be derived reliably from static code scans.
